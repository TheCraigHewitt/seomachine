# Copywriting

Write conversion-focused copy that is clear, specific, and action-oriented.

## Before Writing

Check `.claude/product-marketing-context.md` first. Reuse existing context; ask only for what is missing.

### Gather Inputs

1. Page goal: page type and single primary CTA.
2. Audience: ICP, pain points, objections, buyer language.
3. Offer: product, differentiators, transformation, proof.
4. Traffic context: where visitors come from (ads, organic, email, partner).

---

## Copywriting Principles

- Prefer clarity over cleverness.
- Translate features into outcomes.
- Replace vague claims with specific numbers/examples.
- Mirror customer language from real calls/reviews/tickets.
- Keep one argument per section.
- Do not fabricate proof.

---

## Writing Style Rules

1. Use simple words (`use`, not `utilize`).
2. Use active voice.
3. Cut weak intensifiers (`very`, `really`, `extremely`).
4. Remove empty buzzwords (`innovative`, `seamless`, `robust`) unless proved.
5. State concrete outcomes and timelines.

### Quick QA

- Any jargon a new buyer would not understand?
- Any sentence carrying multiple ideas?
- Any passive construction hiding the actor?
- Any claim without evidence?

For line-by-line sweeps, run **copy-editing** after drafting.

## Page Structure Framework

### Above the Fold

**Headline** -- State the main outcome in one line. Put the strongest value first.

Headline formulas:
- `{Achieve outcome} without {pain}`
- `The {category} for {audience}`
- `Never {unpleasant event} again`

For full formulas, see [references/copy-frameworks.md](references/copy-frameworks.md).
For transition phrasing, see [references/natural-transitions.md](references/natural-transitions.md).

**Subheadline** -- Expand headline with specificity. 1-2 sentences.

**Primary CTA** -- Clear action language. State what they get.

### Core Sections

| Section | Purpose |
|---------|---------|
| Social Proof | Establish trust (logos, metrics, testimonials) |
| Problem/Pain | Show understanding of current friction |
| Solution/Benefits | Connect capabilities to outcomes |
| How It Works | Reduce complexity in 3-4 steps |
| Objection Handling | Resolve risk, timing, and fit concerns |
| Final CTA | Recap value and ask for action |

For detailed page templates, see [references/copy-frameworks.md](references/copy-frameworks.md).

## CTA Copy

**Avoid:** Submit, Learn More, Click Here

**Use:** Start Free Trial, See Pricing for My Team, Create My First Dashboard, Get the Migration Checklist

Formula: `[Action Verb] + [What They Get] + [Qualifier]`

---

## Page-Specific Guidance

| Page Type | Key Guidance |
|-----------|-------------|
| Homepage | Lead with broad value prop, route different intents, one dominant CTA |
| Landing Page | One promise, one CTA, match ad/query message, remove competing nav |
| Pricing | Help users choose quickly, clarify limits, reduce purchase risk |
| Feature | Frame as feature -> benefit -> outcome, add proof |
| About | Explain mission and credibility, link story to buyer value, end with CTA |

---

## Voice and Tone

Set before writing: formality level, brand personality range, technical depth for audience. Keep tone consistent. Increase boldness in headlines, increase clarity in body copy.

---

## Output Format

1. **Page Draft** -- Organized by section with headline, subheadline, body, and CTA text.
2. **Key Rationale** -- Brief explanation of high-impact choices.
3. **Variants** -- 2-3 headline options and 2-3 CTA options.
4. **Meta Copy** (if requested) -- SEO title and meta description.
