# Copy Frameworks Reference

Headline formulas, section patterns, and page structures for conversion pages.

## Headline Formulas

### Outcome-Focused

- **{Achieve outcome} without {pain}** -- "Understand user behavior without drowning in dashboards"
- **{Achieve outcome} by {method}** -- "Generate more leads by seeing which companies visit your site"
- **Turn {input} into {outcome}** -- "Turn support tickets into product insights"
- **{Outcome} in {timeframe}** -- "Launch onboarding emails in 1 day"

### Problem-Focused

- **Never {unpleasant event} again** -- "Never miss a qualified lead again"
- **Stop {pain}. Start {result}.** -- "Stop chasing invoices. Start getting paid on time."
- **{Question tied to core pain}** -- "Tired of manually reconciling reports?"

### Audience-Focused

- **{Category} for {audience}** -- "Revenue analytics for B2B SaaS teams"
- **{Category} for {audience} to {job}** -- "A planning workspace for agencies to run client delivery"
- **You don't need to {hard thing} to {outcome}** -- "You don't need SQL skills to build retention reports"

### Differentiation and Proof

- **The {category} that {differentiator}** -- "The CRM that updates itself"
- **{Number} {people/teams} use {product} to {outcome}** -- "12,000 teams use Acme to ship projects faster"
- **Finally, {category} that {benefit}** -- "Finally, accounting software that doesn't suck"
- **The {adjective} {category} built for {audience}** -- "The lightweight CRM built for startups"
- **What if you could {desirable outcome}?** -- "What if you could close deals 30% faster?"

## Landing Page Section Types

### Core Sections

| Section | Objective |
|---------|-----------|
| Hero | State value prop and primary CTA |
| Social Proof | Establish trust quickly |
| Problem | Validate current pain |
| Benefits | Connect capabilities to outcomes |
| How It Works | Reduce complexity in 3-4 steps |
| Final CTA | Ask for action with risk reduction |

### Supporting Sections

| Section | Use When |
|---------|----------|
| Testimonials | You have specific, attributable proof |
| Case Study Snippet | You can show before/after outcomes |
| Use Cases | Multiple job-to-be-done paths exist |
| FAQ | Repeated objections block conversion |
| Comparison | Buyers evaluate alternatives directly |
| Integrations | Compatibility with existing stack matters |
| Pricing Preview | Decision-makers need cost context early |
| Guarantee | Perceived risk is high |

## Page Structure Templates

### Conversion Narrative (Strong)

```text
1. Hero with clear value proposition
2. Social proof bar
3. Problem framing
4. How it works (3 steps)
5. Key benefits (2-4)
6. Testimonial or case proof
7. Objection handling (FAQ/comparison)
8. Final CTA with risk reducer
```

### Compact Landing Page

```text
1. Hero (headline, subhead, CTA, image)
2. Social proof bar
3. 3 key benefits with icons
4. Testimonial
5. How it works (3 steps)
6. Final CTA with guarantee
```

### Enterprise/B2B Page

```text
1. Hero (business outcome headline)
2. Logo bar (recognizable companies)
3. Business pain section
4. Solution overview
5. Role-based use cases
6. Security/compliance section
7. Integration logos
8. Case study with metrics
9. ROI framing
10. Demo CTA
```

### Product Launch Page

```text
1. Launch hero
2. Demo video/GIF
3. Core improvements (3-5)
4. Before/after comparison
5. Early proof
6. Offer/pricing
7. Urgent CTA
```

## Section Writing Tips

### Problem Section
- Name the specific operational pain.
- Quantify cost (time, money, risk) when possible.
- Transition to solution only after pain is clear.

### Benefits Section
For each benefit: outcome headline, 1-2 sentence mechanism, proof point (metric, quote, or example).

### How It Works
- 3-4 numbered steps. Start each with a verb. End each with user outcome.

### Testimonial Selection
Prefer testimonials with: specific result, before/after context, name + role + company, concrete language over generic praise.
