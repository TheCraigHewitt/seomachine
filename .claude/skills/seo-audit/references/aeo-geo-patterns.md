# AEO and GEO Content Patterns

Reusable blocks for answer engines and AI citation.

## Answer Engine Optimization (AEO) Patterns

### Definition Block

```markdown
## What is [Term]?

[Term] is [one-sentence definition].
[1-2 sentences explaining how it works and why it matters].
```

### Step-by-Step Block

```markdown
## How to [Goal]

[One-line process summary]

1. **[Step]**: [Action and expected output]
2. **[Step]**: [Action and expected output]
3. **[Step]**: [Action and expected output]
```

### Comparison Table Block

```markdown
## [Option A] vs [Option B]

| Criteria | [Option A] | [Option B] |
|----------|------------|------------|
| [Dimension 1] | [Value] | [Value] |
| [Dimension 2] | [Value] | [Value] |
| Best for | [Use case] | [Use case] |

**Bottom line**: [Who should choose which option and why]
```

### Pros and Cons Block

```markdown
## Advantages and Disadvantages of [Topic]

### Pros
- **[Benefit]**: [Specific explanation]
- **[Benefit]**: [Specific explanation]

### Cons
- **[Drawback]**: [Specific explanation]
- **[Drawback]**: [Specific explanation]

**Verdict**: [Balanced conclusion with recommendation]
```

### FAQ Block

```markdown
## Frequently Asked Questions

### [Question as searched]?
[Direct answer in first sentence]. [Brief context].

### [Question as searched]?
[Direct answer in first sentence]. [Brief context].
```

Tips: use natural question phrasing, match "People Also Ask" queries, keep answers 50-100 words.

## Generative Engine Optimization (GEO) Patterns

### Statistic Citation Block

```markdown
[Claim]. According to [Source], [specific statistic + timeframe]. [Why this changes the decision].
```

### Expert Quote Block

```markdown
"[Quote]," says [Expert], [Title] at [Organization]. [One sentence interpreting the quote].
```

### Authoritative Claim Block

```markdown
[Clear claim]. [Source] reports [supporting evidence]. This means [practical implication].
```

### Self-Contained Answer Block

```markdown
**[Question/Topic]**: [Standalone 2-3 sentence answer with concrete detail].
```

### Evidence Sandwich Block

```markdown
[Claim].

Evidence:
- [Data point + source]
- [Data point + source]

[Decision or recommended action].
```

## Domain-Specific Authority Signals

| Domain | Prioritize |
|--------|------------|
| Technology | version numbers, official docs, accurate terminology |
| Health | peer-reviewed citations, credentials, review date |
| Finance | regulator references, dated figures, risk disclaimer |
| Legal | jurisdiction, statute references, professional disclaimer |
| Business/Marketing | case studies, measured outcomes, source attribution |

## Voice Search Pattern

- Start with a direct answer (usually one sentence).
- Use natural question phrasing.
- Keep language conversational and precise.
