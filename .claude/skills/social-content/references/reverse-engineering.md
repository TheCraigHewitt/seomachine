# Reverse Engineering Viral Content

## The 6-Step Framework

### 1. NICHE ID - Find Top Creators
- Pick 10-20 creators with consistent posting and high engagement relative to audience size.
- Prioritize audience overlap with your ICP.

### 2. SCRAPE - Collect Posts at Scale
- Target 500-1000 posts.
- Capture: hook, format, topic, CTA, posting time, engagement metrics.
- Tools: Apify, PhantomBuster, platform exports, or manual sheets.

### 3. ANALYZE - Extract Winning Patterns
- Rank by engagement rate.
- Inspect top 10% for recurring hooks, formats, and CTA styles.
- Compare timing, topic, and post length.

### 4. PLAYBOOK - Codify Reusable Patterns

```text
Pattern: "I [unexpected action] and [surprising result]"
Example: "I stopped posting daily and engagement doubled"
Use case: Contrarian + curiosity

Pattern: "[Number] mistakes that cost [outcome]"
Example: "7 pricing mistakes that cost us $50K"
Use case: Specificity + loss framing

Pattern: "[Strong opinion]"
Example: "Cold outreach is dead"
Use case: Debate trigger
```

Common format patterns:
- Carousel: Hook -> problem -> steps -> CTA
- Thread: Hook -> promise -> delivery -> recap -> CTA
- Story post: Hook -> setup -> conflict -> resolution -> lesson

### 5. LAYER VOICE - Make It Yours
- Write like a trusted peer, not a lecturer.
- Prefer concrete numbers over vague claims.
- Keep sentence rhythm varied and skimmable.
- Lead with real emotion when relevant.

### 6. CONVERT - Turn Attention Into Action
- Soft conversion: newsletter, DM keyword, resource request.
- Direct conversion: context link, demo CTA, case-study CTA.
- Keep conversion ask aligned with post intent.

## The Formula

```text
1. Find what already works.
2. Extract repeatable patterns.
3. Apply your voice and point of view.
4. Test, measure, and iterate.
```

## Reverse Engineering Checklist

- [ ] 10-20 niche creators selected
- [ ] 500+ posts collected
- [ ] Top 10% posts identified
- [ ] Hook patterns documented
- [ ] Format and CTA patterns documented
- [ ] Voice constraints defined
- [ ] Template set created
- [ ] Performance tracking in place
